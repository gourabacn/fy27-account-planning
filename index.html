⚡ Connect Supabase for short links & live responses

Without Supabase, form links are too long to share. It takes 2 minutes to set up — free tier is more than enough.

Step 1 — Create a free project at supabase.com, then run this SQL in the SQL editor:

create table forms (
  id text primary key default substr(md5(random()::text),1,8),
  account_id text not null,
  account_name text not null,
  questions jsonb not null,
  notify_email text,
  created_at timestamptz default now()
);

create table responses (
  id uuid primary key default gen_random_uuid(),
  form_id text references forms(id),
  account_id text,
  data jsonb not null,
  submitted_at timestamptz default now()
);

alter table forms enable row level security;
alter table responses enable row level security;
-- Forms are public (respondents need to read them)
create policy "public read forms" on forms for select using (true);
-- Responses: write-only for the public anon key
-- Reading is done server-side via the Edge Function using the service_role key
create policy "public insert responses" on responses for insert with check (true);

Step 2 — Deploy the get-responses Edge Function (included in the download). Then paste your credentials below.

From Project Settings → API: copy the Project URL and anon key. Choose any strong string as the Admin secret — set it as ADMIN_SECRET in the Edge Function's environment variables (Supabase dashboard → Edge Functions → get-responses → Secrets).
